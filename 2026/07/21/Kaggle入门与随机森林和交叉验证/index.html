Kaggle 入门：用随机森林完成 Titanic 生存预测

Kaggle 的 Titanic 是一个经典的二分类入门项目：根据乘客的船舱等级、性别、同行亲属数量和票价等信息，预测乘客是否生还。它的数据规模不大，却包含了一个完整机器学习项目的主要环节：读取数据、探索数据、处理特征、训练模型、验证效果以及生成提交文件。

本文将使用随机森林建立一个简单、可复现的基线模型，并用分层五折交叉验证评估模型的泛化能力。

1. Kaggle 比赛的基本流程

一个完整的 Kaggle 入门流程通常包括：

  1. 下载比赛提供的 train.csv 和 test.csv；
  2. 从训练集中分离特征 XXX 与标签 yyy；
  3. 清洗缺失值并转换分类变量；
  4. 使用交叉验证评估模型；
  5. 在完整训练集上重新训练模型；
  6. 预测测试集并生成符合要求的 CSV 文件；
  7. 将结果提交到 Kaggle，查看排行榜得分。

训练集中的 Survived 是需要预测的标签：

  • 0：未生还；
  • 1：生还。

测试集没有 Survived 列，因为这正是参赛者需要提交的预测结果。

接下来设置随机森林模型：

随机森林是由多棵决策树组成的集成模型。每棵树会在随机抽取的样本和部分特征上学习，最终通过多数投票得到分类结果。相比单棵决策树，它通常更稳定，也不容易因训练数据中的偶然噪声而严重过拟合。

这里三个参数的作用是：

  • n_estimators=100：构建 100 棵决策树；
  • max_depth=5：限制每棵树的最大深度，控制模型复杂度；
  • random_state=42：固定随机种子，使实验结果可以复现。

5. 使用分层五折交叉验证

如果只进行一次训练集与验证集划分，模型分数可能会受到这次随机划分的明显影响。五折交叉验证会将训练数据分成五份：每次使用其中四份训练、剩余一份验证，共进行五轮，最后统计五次得分的均值和标准差。

Titanic 是分类问题，因此使用 StratifiedKFold。它会尽量让每一折中的生还与未生还比例接近原始数据，评估结果通常比普通随机划分更稳定。

需要注意：交叉验证本身不会提高模型准确率。它的作用是更可靠地评估模型，并帮助我们比较特征、算法和参数。只有根据验证结果调整方案，模型性能才可能真正提升。

7. 还能怎样改进

这个模型的目标是快速跑通 Kaggle 流程，并不是最终方案。后续可以从以下方向继续尝试：

  1. 加入 Age 和 Embarked，并分别处理数值缺失值与分类缺失值；
  2. 从姓名中提取称谓，如 Mr、Mrs、Miss 和 Master；
  3. 构造家庭规模 FamilySize = SibSp + Parch + 1；
  4. 从 Cabin 中提取甲板编号，或构造“是否有船舱记录”的特征；
  5. 使用网格搜索或随机搜索调整树的数量、深度、叶节点样本数等参数；
  6. 除准确率外，同时观察混淆矩阵、精确率、召回率和 F1 分数。

8. 总结

通过 Titanic 项目，我们完成了一个机器学习竞赛的最小闭环：

随机森林为结构化表格数据提供了一个易用而可靠的基线；分层交叉验证则让模型评估不再依赖某一次偶然的数据划分。掌握这套流程后，就可以继续尝试更丰富的特征工程和模型调参，并用同一套验证方式判断改进是否有效。
