一.计算机系统概论

1.1 计算机系统的软硬件概念

  • 硬件：指计算机系统中实际存在的物理设备，如CPU、内存、硬盘、显示器等。
  • 软件：指计算机系统中运行的程序和数据，如操作系统、应用程序、数据库等。

"软件"的地位和作用已经成为评价计算机系统性能好坏的重要指标。
计算机软件可以分为两大类：系统软件和应用软件。

  • 系统软件：指操作系统、编译程序、汇编程序等，它们是计算机系统的基础，提供了计算机系统的运行环境和服务。
  • 应用软件：指用户在计算机上进行特定任务的程序，如文字处理、 spread sheet、数据库管理系统等。

1.2 计算机系统的层次结构

虚拟机器M4(高级语言机器):

  • 它是一种能够直接执行高级语言程序的机器。
  • 高级语言程序是一种人可读的程序，而机器指令是一种机器可读的程序。
  • 高级语言机器通过编译程序将高级语言程序转换为汇编语言程序或直接机器语言程序，然后执行这些程序。
    |
    |
    虚拟机器M3(汇编语言机器):
  • 它是一种能够直接执行汇编语言程序的机器。
  • 汇编语言程序是一种人可读的程序，而机器指令是一种机器可读的程序。
  • 汇编语言机器通过汇编程序将汇编语言程序转换为机器指令程序，然后执行这些程序。
    |
    |
    虚拟机器M2(操作系统机器):
  • 它是一种能够直接执行操作系统程序的机器。
  • 操作系统程序是一种人可读的程序，而机器指令是一种机器可读的程序。
  • 操作系统机器通过操作系统将应用程序转换为机器指令程序，然后执行这些程序。
    |
    |
    虚拟机器M1(机器语言机器):
  • 它是一种能够直接执行机器指令程序的机器。
  • 机器指令程序是一种机器可读的程序，而机器指令是一种机器可读的程序。
  • 机器语言机器通过直接执行机器指令程序来完成计算任务。
    |
    |
    微程序机器M0(微指令系统):
  • 它是一种能够直接执行微指令程序的机器。
  • 微指令程序是一种机器可读的程序，而微指令是一种机器可读的程序。
  • 微程序机器通过直接执行微指令程序来完成计算任务。

机器语言、汇编语言、高级语言的区别:
1.高级语言：是一种指令集的体系，可以跨平台使用，语法和结构更类似汉字或者普通英文，且由于远离对硬件的直接操作，使得一般人更容易学习。
2、汇编语言：是一种用于电子计算机、微处理器、微控制器或其他可编程器件的低级语言。汇编语言对应着不同的机器语言指令集，特定的汇编语言和特定的机器语言指令集是一一对应的，不同平台之间不可直接移植。
3、机器语言：由0、1构成，唯一不经翻译即可为机器直接理解和接受的程序语言或指令代码。
机器语言、汇编语言、高级语言的联系：
高级语言编译成机器语言，机器语言通过汇编转换成机器指令后在计算机上执行。 汇编指令和机器语言完全对应，是机器语言中0、1代码的助记符表示形式。

1.3 计算机组成和计算机体系结构

计算机体系结构：

  • 它是指计算机系统的外部结构，包括硬件和软件的组织和交互方式。
  • 计算机体系结构包括指令集、数据表示、存储系统、输入输出系统等。
  • 不同的计算机体系结构有不同的性能和功能。

计算机组成：

  • 是指如何实现计算机体系结构所体现的属性。
  • 它是指计算机系统的内部结构，包括硬件和软件的组织和交互方式。
  • 计算机组成包括CPU、内存、存储系统、输入输出系统等。
  • 不同的计算机组成有不同的性能和功能。

例子：指令系统体现了机器的属性，这是属于计算机结构的问题。但指令的实现如加法指令、减法指令等，是属于计算机组成的问题。

1.4 计算机的基本组成

1.4.1 冯诺依曼计算机的特点

  1. 计算机由五大部件组成：运算器、控制器、存储器、输入设备、输出设备
  2. 指令和数据以同等地位存于存储器，可按地址寻访
  3. 指令和数据用二进制表示
  4. 指令由操作码和地址码组成
  5. 指令顺序存储
  6. 以运算器为中心

1.5 计算机的性能指标

1.5.1 字长

  • 机器字长（CPU 字长）：CPU 一次能处理的二进制数据位数，通常等于 ALU/通用寄存器位数；会影响数据总线宽度与主存访问粒度。
  • 指令字长：一条机器指令的二进制代码总位数。
  • 存储字长：存储器一个存储单元可存放的二进制位数。

1.5.2 存储容量

  • 存储容量：存储系统可存放的二进制信息总量（按 bit 或 Byte 计）。
  • 常用表达：
    • 按 bit：Cbit=N×LC_{bit} = N \times LCbit​=N×L
    • 按 Byte：CByte=N×L8C_{Byte} = \frac{N \times L}{8}CByte​=8N×L​
  • 其中 NNN 为存储单元个数，LLL 为存储字长（bit）。

1.5.3 存储器带宽

  • 存储器带宽：单位时间内存储器能传输的数据量（数据传输速率）。
  • 常见写法：
    • 按“字/秒”：BWword=1TcycleBW_{word} = \frac{1}{T_{cycle}}BWword​=Tcycle​1​
    • 按“Byte/秒”：BWByte=L/8TcycleBW_{Byte} = \frac{L/8}{T_{cycle}}BWByte​=Tcycle​L/8​
  • 其中 TcycleT_{cycle}Tcycle​ 是存储周期（一次读/写所需时间），LLL 为存储字长（bit）。

1.5.4 运算速度（性能指标）

  • 主频 fff：CPU 时钟频率（Hz），时钟周期 T=1/fT = 1/fT=1/f。
  • CPI（Cycles Per Instruction）：平均每条指令所需时钟周期数

1.6.1 流程图

1.6.2 各部件功能

  • ALU（算术逻辑单元） ：执行加/减/与/或/比较等运算，输出结果供寄存器写回。
  • ACC（累加器） ：运算器的核心工作寄存器，常作为 ALU 的一个输入与结果暂存寄存器（很多指令默认以 ACC 为隐含操作数/目的）。
  • MQ（乘商寄存器） ：用于乘法/除法等运算时暂存乘数、商或中间结果（做加法/存数时通常不参与）。
  • X（通用/暂存寄存器） ：用于暂存从主存读出的操作数，作为 ALU 的另一输入（图中加法时与 ACC 同时送入 ALU）。
  • CU（控制单元） ：发出控制信号，协调“取指、译码、执行”的时序（控制读/写、寄存器装入、PC 自增等）。
  • IR（指令寄存器） ：存放当前正在执行的指令（操作码+地址码），译码时 CU 读取 IR 的内容。
  • PC（程序计数器） ：存放下一条将要取出的指令地址；取指后通常自增（或被跳转类指令改写）。
  • MAR（存储器地址寄存器） ：存放将要访问的主存地址（读/写主存都先把地址送入 MAR）。
  • MDR（存储器数据寄存器） ：主存数据缓冲寄存器；读主存时接收数据，写主存时暂存待写数据。
  • 存储体（主存） ：按地址存放指令和数据（冯诺依曼：指令/数据同存储体）。

1.6.3 工作步骤

  • 1：PC → MAR （把要取的指令地址送去）
  • 2：MAR → 存储体（地址线/寻址） （用 MAR 中的地址去访问主存单元）
  • 3：存储体 → MDR （从主存读出“指令/数据”到 MDR）
  • 4：PC ← PC + 1 （PC 自增，指向下一条指令）
  • 5：MDR → IR （把取到的指令装入 IR）
  • 6：IR（地址码）→ MAR （执行阶段：把操作数地址送去 MAR）
  • 7：ACC → MDR （存数时：把要写回主存的数据先放入 MDR）
  • 8：MDR → 存储体（写） （把 MDR 的数据写入主存指定地址）
  • 9：MDR → X （加法时：把读到的操作数送入 X）
  • 10：ACC → ALU （ALU 输入1）
  • 11：X → ALU （ALU 输入2；你标注“10 和 11 同时”就是双输入并行送入）
  • 12：ALU → ACC （ALU 结果写回 ACC）

存数指令 1,2,3,4,5,6,2,7,8

  • 1-5 ：取指（PC 给地址→主存读→IR 装入→PC 自增）
  • 6,2 ：把指令的地址码送 MAR 并寻址到目标单元
  • 7,8 ：把 ACC 的数据经 MDR 写回主存（完成 store）

加法指令 1,2,3,4,5,6,2,3,9,10,11(同时),12

  • 1-5 ：取指
  • 6,2,3,9 ：把操作数地址送 MAR，主存读出到 MDR，再送入 X
  • 10,11,12 ：ACC 与 X 同时送 ALU 相加，结果写回 ACC（完成 add）

三.系统总线

3.1 总线的基本概念

  • 计算机各功能部件之间需要进行信息传递，因此需要信息传输线路。
  • 连接方式主要有两类：
    1. 专用线路：在需要传递信息的部件之间建立专用连接。
    2. 总线方式：建立公共信息传输通路（总线），多个功能部件共享该通路完成信息传送。
  • 总线方式的特点：
    • 任意两个功能部件间的信息传送通常都要经过总线。
    • 同一时刻总线一般只能被一对部件占用，其他部件采用分时方式排队使用（涉及总线仲裁/排队规则）。
    • 连接更简化：各部件只需提供与总线的接口，无需为每一对部件分别设计连接线路。
    • 扩展/更换更方便：新增或替换部件时，只要通过接口接入/卸下总线即可。
  • 总线（Bus）的定义：计算机系统中多个功能部件之间进行信息传送的公共通路。
  • 总线上常见传送信息包括：地址信息、控制信息、数据信息，对应地址线、控制线、数据线；另外还会有电源线与接地线等支持线路。
